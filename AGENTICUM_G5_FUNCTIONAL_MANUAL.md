# AGENTICUM G5 — Funktionshandbuch (Blueprint & Prozeduren)

## 1. BEDIENUNG DER GENIUS CONSOLE

- **Input**: Freitext-Briefings oder Voice-Commands.
- **Interpretation**: Der SN-00 Orchestrator übersetzt den Intent in einen Workflow-Graphen.
- **Monitoring**: Echtzeit-Logs und Agent-Status-Visualisierung.

## 2. WORKFLOW BLUEPRINTS

### 2.1 Pillar Content Forge (CC-06 + SP-01)

1. **Trigger**: Keyword oder Trend-Alert.
2. **Research**: Grounding via Google Search & Vault Indexing.
3. **Strategy**: SP-01 definiert H2-Struktur und SEO-Angriffspunkte.
4. **Forge**: CC-06 generiert 2000+ Worte Pillar-Text.
5. **Audit**: RA-01 prüft Fakten und Marken-Compliance.

### 2.2 Creative Studio (DA-03)

1. **Prompt**: Visual Concept.
2. **Forge**: Imagen 3 generiert 4K Assets.
3. **Persistence**: Speicherung im Vault mit Metadaten.
4. **Distribution**: Bereitstellung für Ads/Social.

## 3. SECURITY SENATE (Compliance)

- **Zero Veto Protocol**: Content wird blockiert, wenn Risikoscore > 0.7.
- **Iterative Revision**: Feedback-Loop vom Senate zurück an den Agenten.

## 4. VAULT & MEMORY

- **Persistence**: Alle Assets (JSON, MD, JPG) werden im `/data/vault` indiziert.
- **RAG Capability**: Agenten nutzen Vault-Inhalte als primäre Wissensquelle.
